# CLAUDE.md — Project Instructions

## What This Is

"The Architecture of Wonder" by Chris Michael. A non-fiction book teaching behavioral profiling, audience psychology, and performance craft through the lens of mentalism. 40 chapters, 8 parts. The book's design demonstrates its own content — the Meta Reveal chapter shows the reader what was done to them.

## The Source of Truth

**`Architecture-of-Wonder.docx`** is the master document. Everything else is generated from it.

- `manuscript-extracted.txt` — plain text extracted from the DOCX (intermediate file)
- `Architecture-of-Wonder-DESIGNED.html` — fully designed HTML output (generated by build script)
- **Never edit the HTML or extracted text directly** — changes will be overwritten on the next build

## How to Edit the Book

### Text edits (prose, new sections, restructuring)
Edit the DOCX programmatically using Python's `zipfile` and `xml.etree.ElementTree`:
```python
import zipfile, xml.etree.ElementTree as ET
# Always register namespaces before writing
ET.register_namespace('w', 'http://schemas.openxmlformats.org/wordprocessingml/2006/main')
ET.register_namespace('r', 'http://schemas.openxmlformats.org/officeDocument/2006/relationships')
```
- Always back up to `backups/` before major edits
- Find insertion points by searching for known text, not by remembered positions
- After inserting/removing elements, re-read `list(body)` before using position-based indexing
- Use `xml:space="preserve"` on all text elements

### Adding figures/photos
1. Image file goes in `resources/metv-images/`
2. Embed in the DOCX using OOXML inline drawing XML (so it's visible in Word/Google Docs)
3. Add entry to the `FIGURES` dict in `build-book.py` (key format: `"CHAPTER <num>:<section header>"`)
4. The build script injects the `<img>` tag after the matching section header
5. Track rights status in the `rights` field of each FIGURES entry

### After any edit
```bash
# Step 1: Extract text
python3 -c "
import zipfile, xml.etree.ElementTree as ET
with zipfile.ZipFile('Architecture-of-Wonder.docx', 'r') as z:
    doc_xml = z.read('word/document.xml')
ns = {'w': 'http://schemas.openxmlformats.org/wordprocessingml/2006/main'}
root = ET.fromstring(doc_xml)
body = root.find('.//w:body', ns)
with open('manuscript-extracted.txt', 'w') as f:
    for p in body.findall('.//w:p', ns):
        text = ''.join(r.text or '' for r in p.findall('.//w:r/w:t', ns))
        f.write(text + '\n')
"

# Step 2: Build designed HTML
python3 build-book.py

# Step 3: Build gated version (ALWAYS run this after build-book.py)
python3 build-gated.py
```

## The Build Script (`build-book.py`)

Converts extracted text into fully designed HTML. All design configuration lives here:
- `HOOK_LINES` — chapter openers (one per chapter)
- `KEY_READS` — chapter closers (one per chapter)
- `WHAT_YOU_JUST_DID` — meta-reveal moments at specific chapters
- `PATTERN_INTERRUPTS` — statistical callout boxes
- `FIGURES` — image placement map (chapter + section header -> image data)
- `FIVE_CS_HTML` — injected at Chapter 7

**Chapter numbering note:** The parser's chapter numbers (used in config dicts) can differ from the TOC numbers in the HTML. Always verify with the parser if unsure.

## Chris's Voice

When writing or editing prose, match this voice:
- **Direct.** Talks TO the reader, not AT them. Second person. Conversational authority.
- **Grounded.** Every concept gets a sensory picture, not an abstraction.
- **Punchy.** Short declarative sentences land between longer analytical paragraphs.
- **Stories have setting, stakes, and a clock.** Not vague anecdotes.
- **Authority without arrogance.** States and qualifies, never waffles.
- **Willing to break form.** Parenthetical humor, editor notes, direct reader instructions.

**Do NOT:**
- Hedge with "perhaps" or "it could be argued"
- Use academic passive voice
- Add emojis or exclamation marks
- Over-explain — trust the reader
- Add commentary that reads as academic summary

**Structural pattern:** myth -> definition -> foundation -> framework -> application -> caution

**Sentence rhythm:** Long analytical paragraph -> Short declarative punch -> Long again
